# Reglas globales del proyecto

## Principios obligatorios
- **KISS**: Mantén el código directo, simple y sin complejidad accidental.
- **DRY**: Evita duplicar estructuras, lógica y funciones. Extrae solo cuando aporte valor real.
- **YAGNI**: No agregues lógica, abstracciones o capas que no se necesitan hoy.

## Estilo Clean Code
- El código debe ser **autoexplicativo**. Nombres claros > comentarios.
- **Evita comentarios innecesarios**; se permiten solo cuando la lógica es realmente compleja o no es evidente.
- Prefiere **renombrar funciones/variables** antes que explicar su propósito con comentarios.
- Si un comentario explica “qué hace”, elimínalo: el código debe responder esa pregunta por sí mismo.
- Se permiten comentarios para:
  - decisiones de arquitectura,
  - aclaraciones de negocio,
  - comportamientos no obvios,
  - advertencias o limitaciones conocidas.
